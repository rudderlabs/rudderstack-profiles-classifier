Syntax for vars definition has been modified.:
In the list of vars, to refer to an entity-var, use {{ <entity_name>.Var('entity_var_name') }}. 
To refer to an input-var, use {{ <associated_input_model_name>.Var('input_var_name') }}.
In the auto-migration, we try to grep the var references and convert them to the newer dot var syntax.
But it is recommended to check the migrated file once as regex operations may cause issues (for example some column name is same as a var-name).
Below is an example of old style entity-var and the expected new syntax entity var.
vars:
  - entity_var:
		name: a_num_min
		select: min(num_a)
		from: inputs/tbl_a
  - entity_var:
		name: a_range
		select: "{{ subtract_range('a_max', 'a_num_min') }}"
It will be changed to following syntax:.
vars:
  - entity_var:
		name: a_num_min
		select: min({{tbl_a.Var("num_a")}})
		from: inputs/tbl_a
  - entity_var:
		name: a_range
		select: '{{ subtract_range("{{user.Var(\'a_max\')}}", "{{user.Var(\'a_num_min\')}}") }}'
